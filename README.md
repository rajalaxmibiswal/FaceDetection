# 🎭 Face Detection & Recognition System

A real-time face detection and recognition project built using **OpenCV**, **Keras (TensorFlow)**, and a **CNN (Convolutional Neural Network)**. The system can collect face images, train a model on them, and then recognize faces in real time from an IP camera stream.

---

## 📁 Project Structure

```
FaceDetection/
│
├── haarcascade_frontalface_default.xml   # Pre-trained face detector (Haar Cascade)
├── final_model.h5                        # Trained CNN model (generated after training)
│
├── collect_data.py                       # Step 1 – Capture face images from IP camera
├── consolidated_data.py                  # Step 2 – Process images and create dataset
├── train_model.py                        # Step 3 – Train CNN model on the dataset
├── recognize.py                          # Step 4 – Real-time face recognition
│
├── images/                               # Folder where collected face images are saved
│   └── (auto-generated by collect_data.py)
│
└── data/                                 # Folder for processed dataset files
    ├── images.p                          # Pickled image array (auto-generated)
    └── labels.p                          # Pickled labels array (auto-generated)
```

## 📱 IP Camera Setup

This project reads video from an **IP camera** (e.g., using the **IP Webcam** app on Android).

1. Install the **IP Webcam** app on your Android phone.
2. Open the app → tap **"Start Server"**.
3. Note the URL shown on screen, e.g.: `http://192.168.x.x:8080`
4. The image endpoint will be: `http://192.168.x.x:8080/shot.jpg`
5. Update this URL in both `collect_data.py` and `recognize.py`:

```python
url = "http://<YOUR_PHONE_IP>:8080/shot.jpg"
```

> 💡 Make sure both your PC and phone are connected to the **same Wi-Fi network**.

## 🚀 Step-by-Step Execution Guide

### ✅ Step 1 — Collect Face Data (`collect_data.py`)

**Purpose:** Captures 100 face images of one person at a time from the IP camera and saves them to the `images/` folder.

```bash
python collect_data.py
```

**What it does:**
- Connects to the IP camera stream.
- Uses Haar Cascade to detect faces in each frame.
- Crops and saves the face region as a `.jpg` image.
- Shows a live counter on screen (e.g., `45/100`).
- Once 100 images are captured, asks you to enter the person's name.
- Saves images as: `images/<name>_0.jpg`, `images/<name>_1.jpg`, ... `images/<name>_99.jpg`
- Press **`q`** to quit early (less than 100 images won't be saved).

> 🔁 **Repeat this step for each person** you want to recognize (e.g., bottle, Rose, smile_face).

**Before running**, make sure:
- The `images/` folder exists: `mkdir images`
- The Haar Cascade XML path inside the script is correct (update if needed):
  ```python
  classifier = cv2.CascadeClassifier('haarcascade_frontalface_default.xml')
  ```

---

### ✅ Step 2 — Consolidate Data (`consolidated_data.py`)

**Purpose:** Reads all images from the `images/` folder, resizes and converts them to grayscale, and saves them as a NumPy dataset for training.

```bash
python consolidated_data.py
```

**What it does:**
- Reads every image file from `images/`.
- Resizes each image to **100×100 pixels**.
- Converts each image to **grayscale**.
- Extracts the **label (class name)** from the filename (the part before the first `_`).
  - e.g., `Rose_42.jpg` → label = `Rose`
- Saves the image array and label array as pickle files:
  - `data/images.p`
  - `data/labels.p`
- Displays one sample image using matplotlib.

**Before running**, make sure:
- The `data/` folder exists: `mkdir data`

---

### ✅ Step 3 — Train the Model (`train_model.py`)

**Purpose:** Loads the processed dataset and trains a CNN model to classify faces.

```bash
python train_model.py
```

**What it does:**
- Loads `images.p` and `labels.p` from the `data/` folder.
- Normalizes pixel values to the range `[0, 1]` (divides by 255).
- Reshapes images to `(num_samples, 100, 100, 1)` for the CNN.
- Encodes string labels to integers, then to one-hot vectors.
- Builds a CNN with the following architecture:

```
Input (100x100x1)
  → Conv2D(32 filters, 3x3, ReLU)
  → MaxPooling2D(2x2)
  → Conv2D(64 filters, 3x3, ReLU)
  → MaxPooling2D(2x2)
  → Flatten
  → Dense(128, ReLU)
  → Dense(num_classes, Softmax)
```

- Trains for **10 epochs** with **batch size 16** using the Adam optimizer.
- Saves the trained model as `final_model.h5`.

> ⏳ Training time depends on the size of your dataset and system hardware.

---

### ✅ Step 4 — Real-Time Recognition (`recognize.py`)

**Purpose:** Uses the trained model to detect and classify faces in real-time from the IP camera.

```bash
python recognize.py
```

**What it does:**
- Loads the trained `final_model.h5`.
- Loads the Haar Cascade face detector.
- Continuously reads frames from the IP camera.
- For each frame:
  - Detects faces using Haar Cascade.
  - Draws a **blue rectangle** around detected faces.
  - Preprocesses the face (grayscale → resize → histogram equalization → normalize).
  - Feeds the face into the CNN model.
  - Displays the **predicted label** on the frame.
- Press **`q`** to quit.

> ⚠️ Update the class labels in `recognize.py` to match your own trained classes:
> ```python
> def get_pred_label(pred):
>     labels = ["bottle", "Rose", "smile_face"]  # Change these to your class names
>     return labels[pred]
> ```

## 🧠 How the CNN Works (Simple Explanation)

| Layer | What it does |
|---|---|
| **Conv2D** | Scans the image for features like edges, curves, and patterns |
| **MaxPooling2D** | Reduces image size while keeping the most important features |
| **Flatten** | Converts the 2D feature map into a 1D vector |
| **Dense (128)** | Learns complex combinations of features |
| **Dense (softmax)** | Outputs probabilities for each class |

The model learns to distinguish faces by training on many examples of each person/object.

## 🔧 Common Issues & Fixes

| Problem | Fix |
|---|---|
| Camera not connecting | Check the IP address and ensure phone & PC are on same Wi-Fi |
| `haarcascade_frontalface_default.xml` not found | Use the full path or place the file in the same directory |
| `data/` or `images/` folder not found | Create them manually: `mkdir data images` |
| Low accuracy | Collect more images (200+), ensure good lighting, vary angles |
| Model not found | Run `train_model.py` before `recognize.py` |
| Labels mismatch | Update `get_pred_label()` in `recognize.py` to match your class names |

---

## 📌 Notes

- The label is extracted from the **filename prefix** before the first underscore (`_`). Always name images as `classname_number.jpg`.
- The model uses **grayscale** images. Color information is not used.
- **Histogram equalization** (`cv2.equalizeHist`) is applied during recognition to normalize lighting conditions.
- The Haar Cascade is only used for **face localization** — the CNN handles **classification**.
